# Test-Driven Development Command
# Use: /test <feature/scenario>

## Purpose
Write tests first, then make them pass. Red-Green-Refactor cycle.

## Usage
```
/test Add tests for payment validation
/test Cover the webhook handler
/test Test the discount calculation
```

## TDD Cycle
1. **RED** - Write failing test
2. **GREEN** - Write minimal code to pass
3. **REFACTOR** - Improve code while keeping tests green

## Principles
- Test behavior, not implementation
- Each test is independent
- Descriptive test names
- Arrange-Act-Assert pattern
- Cover edge cases and error paths
