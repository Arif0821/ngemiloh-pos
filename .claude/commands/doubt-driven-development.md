# doubt-driven-development

Use the `doubt-driven-development` skill.

Adversarial fresh-context review of every non-trivial decision in-flight - CLAIM → EXTRACT → DOUBT → RECONCILE → STOP, with optional user-authorized cross-model escalation.

Use when stakes are high (production, security, irreversible), working in unfamiliar code, or a confident output is cheaper to verify now than to debug later.
